# AI/MCP安全

## AI应用安全测试（五大领域）

### 1. 应用层攻击
| 攻击 | 描述 |
|------|------|
| 直接Prompt注入 | 覆盖系统指令 |
| 间接Prompt注入 | 通过外部数据源注入 |
| CoT干扰 | 干扰思维链推理 |
| Agent滥用 | 滥用工具调用能力 |
| 代码执行突破 | 沙箱逃逸 |
| Memory投毒 | 污染长期记忆 |

### 2. MCP与Agent风险
| 风险 | 描述 |
|------|------|
| 工具描述投毒 | 在工具描述中嵌入恶意指令 |
| 指令覆盖 | 覆盖系统提示词 |
| 隐藏指令注入 | 不可见字符注入 |
| 未授权资源访问 | 越权访问文件/API |
| Skills/Rules供应链 | 恶意技能文件 |

### 3. 身份与授权
- 动作滥用、角色逃逸、权限漂移
- 云凭据滥用（AWS/GCP/Azure key泄露）

### 4. 数据与隐私
- Prompt泄漏（系统提示词泄露）
- 敏感数据暴露
- 训练数据问题
- 模型逆推
- API数据窃取

### 5. 基线与部署
- CI/CD缺陷
- 容器逃逸
- 向量数据库安全
- 沙箱失效
- 环境隔离缺陷

## GAARM风险矩阵

GAARM = Governance, Authentication, Authorization, Risk Management

用于评估AI系统的安全风险等级：
- **G**overnance: 治理策略、合规要求
- **A**uthentication: 身份认证机制
- **A**uthorization: 权限控制模型
- **R**isk Management: 风险评估与缓解

## MCP安全测试要点

```
1. 工具描述审计：检查是否有隐藏指令
2. 输入验证：测试边界值和特殊字符
3. 权限检查：验证最小权限原则
4. 数据流追踪：敏感数据是否泄露
5. 供应链验证：第三方工具的安全性
